## 📌 مشکل

صفحه Claude.ai به‌صورت پیش‌فرض از RTL (راست‌به‌چپ) پشتیبانی نمی‌کند. این باعث می‌شود متن‌های فارسی، عربی و عبری ناخوانا و چپ‌چین نمایش داده شوند — حتی وقتی Claude جواب‌هایش را به درستی به فارسی می‌نویسد.

## ✅ راه‌حل

این افزونه به‌صورت هوشمند متن‌های RTL را شناسایی و راست‌چین می‌کند — بدون اینکه متن‌های انگلیسی را تحت تأثیر قرار دهد.

---

## ✨ ویژگی‌ها

- 🔍 **تشخیص خودکار زبان** — فقط پاراگراف‌هایی که واقعاً فارسی/عربی/عبری هستند RTL می‌شوند
- 🔤 **فونت Vazirmatn** — فونت فارسی باکیفیت برای خوانایی بهتر
- ↔️ **رفع مشکل Bidi** — کلمات انگلیسی داخل متن فارسی با ایزوله‌سازی Unicode (LRI/PDI) درست نمایش داده می‌شوند
- ✍️ **باکس ورودی** — هنگام تایپ فارسی، خود باکس نوشتن هم راست‌چین می‌شود
- ⬇️ **Export فارسی** — خروجی Markdown و PDF با حفظ فرمت RTL
- 🌐 **چندزبانه** — پشتیبانی از فارسی، عربی و عبری
- ⌨️ **میانبر کیبورد** — `Alt+R` برای فعال/غیرفعال کردن سریع
- 🔄 **همگام‌سازی چند تب** — تغییر state در یک تب روی همه تب‌های Claude اعمال می‌شود
- ⚡ **بهینه** — استفاده از debounced MutationObserver برای عملکرد روان

## 🚀 نصب (Developer Mode)

تا زمان انتشار در Chrome Web Store، از این روش استفاده کنید:

1. این مخزن را دانلود یا clone کنید:

   ```bash
   git clone https://github.com/yourusername/claude-rtl.git
   ```

2. در Chrome آدرس `chrome://extensions` را باز کنید

3. گزینه‌ی **Developer mode** را از گوشه‌ی بالا-راست فعال کنید

4. روی **Load unpacked** کلیک کنید و پوشه‌ی پروژه را انتخاب کنید

5. وارد [claude.ai](https://claude.ai) شوید و از آیکون افزونه استفاده کنید

## 🔧 نحوه‌ی کارکرد

### تشخیص زبان

افزونه نسبت کاراکترهای RTL (فارسی/عربی/عبری) به کل حروف هر المان را محاسبه می‌کند. اگه این نسبت بالای ۳۰٪ باشد، آن المان RTL می‌شود — این یعنی پاراگراف‌های کاملاً انگلیسی هرگز تحت تأثیر قرار نمی‌گیرند.

### رفع مشکل Bidi

برای متن‌های مخلوط مثل:

> «منطق: Stochastic RSI روی RSI اعمال می‌شود»

کلمات انگلیسی با کاراکترهای کنترلی Unicode (LRI: `U+2066` / PDI: `U+2069`) ایزوله می‌شوند تا مرورگر ترتیب صحیح نمایش را بداند.

### MutationObserver بهینه

به جای اسکن کل DOM در هر تغییر، فقط nodeهای جدید پردازش می‌شوند. برای متن‌های در حال استریم شدن از debounce 1000ms استفاده می‌شود.
